# System Specifications

This directory contains cross-cutting system specifications for your project, generated by the `/spec-requirements` skill in **System Spec Mode**.

## When to use System Specs

Unlike feature-specific specs (which live inside feature branches under `artifacts/features/<slug>/`), system specs document durable, cross-cutting requirements, policies, or system-wide behaviors. Examples include:
- API authorization protocols
- Common database models
- Security compliance rules
- Custom layout design systems

## Workflow

1. Run `/spec-requirements` with the `System Spec` risk flag or mode.
2. The agent will author the spec and output it directly to `docs/system-specs/<name>.md` using `references/system-spec-template.md`.
3. These specs are committed to the main branch and serve as durable references for future feature planning and implementation phases.
