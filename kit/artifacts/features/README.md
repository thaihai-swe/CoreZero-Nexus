# Feature Artifacts

Each feature gets a subdirectory under `artifacts/features/<slug>/`. Below are all standard artifacts, their owning skill, phase of creation, and schema reference.

| Artifact | Owning skill | Phase | Schema |
|---|---|---|---|
| `status.md` | spec-research / all | All | `_shared/status-template.md` |
| `analysis.md` | spec-research | Researching | `spec-research/references/analysis-template.md` |
| `proposal.md` | spec-requirements | Spec'ing | `spec-requirements/references/proposal-template.md` |
| `spec.md` | spec-requirements | Spec'ing | `spec-requirements/references/spec-template.md` |
| `plan.md` | spec-plan | Planning | `spec-plan/references/plan-template.md` |
| `tasks.md` | spec-plan | Planning | `spec-plan/references/tasks-template.md` |
| `progress.md` | spec-implement / context-session | Implementing | `context-session/references/progress-template.md` |
| `session-extracts.md` | spec-implement / context-session | Implementing+ | `context-session/references/session-extracts-template.md` |
| `review.md` | harness-verify / code-review | Verifying | `harness-verify/references/review-template.md` |
| `testing-scenarios.md` | harness-verify | Verifying | `harness-verify/references/testing-scenarios-template.md` |
| `promotions.md` | context-memory | Memory Sync | (proposal, free-form) |
| `memory-audit.md` | context-memory --audit | Memory Sync | (audit report) |
