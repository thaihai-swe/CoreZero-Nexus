## Lightweight ADR Template

Use this shorter format for minor, low-ceremony, but worth-recording architectural decisions:

```markdown
# ADR-[number]: [Title] (Lightweight)

Status: Proposed | Accepted | Deprecated | Superseded
Date: [Date]
Feature slug: [slug]
Related spec: [link to spec.md]

### 1. Decision
[One sentence specifying the choice made.]

### 2. Context
[What problem does this address? What forces are at play?]

### 3. Options Considered
- Option A: [Summary]
- Option B: [Summary]

### 4. Rationale
[Why did we choose the selected option? Note any key trade-offs.]

### 5. Consequences
[What becomes easier or harder as a result of this decision?]
```
