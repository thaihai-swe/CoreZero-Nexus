# Coverage Checklist

Use this checklist before finalizing a system flow document.

## Core Coverage

- Audience is explicit
- System boundary is explicit
- Main entry points are identified
- Primary end-to-end flow is documented
- Core workflows are listed and explained

## Structural Coverage

- Major components are named
- Responsibility boundaries are clear
- External dependencies are identified
- Important data or event handoffs are documented

## Visual Coverage

- One primary Mermaid diagram is present
- One supporting diagram or structured section is present for non-trivial systems
- Diagrams are focused and readable

## Trust Coverage

- Facts are traceable to source
- Unknowns are labeled
- Conflict between code and prior docs is noted
- The document is useful without reading implementation code first
